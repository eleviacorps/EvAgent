# Audience Adaptation

## Steps
1. Define audience: demographics, expertise level, goals, pain points, preferred channels
2. Adjust vocabulary: technical terms for experts, plain language for general audiences
3. Adjust depth: executive summaries for busy readers, thorough analysis for specialists
4. Adjust tone: urgent for sales, reassuring for support, inspiring for thought leadership
5. Adjust examples and analogies: relevant to the audience's domain and experience
6. Adjust format: long-form for white papers, scannable for web, concise for email

## Examples
- Same topic, different audiences:
  - CTO: architecture, scalability, security, compliance, ROI
  - Developer: code examples, API docs, integration steps, debugging tips
  - Business user: features, benefits, use cases, case studies, pricing
- Executive summary: 3 bullet points + one-page overview
- Technical deep-dive: full specifications with diagrams and code

## Anti-patterns
- Writing for yourself instead of your audience
- Dumbing down without respecting the reader's intelligence
- Using jargon with non-specialist audiences
- Assuming too much context (don't assume they know your product)
