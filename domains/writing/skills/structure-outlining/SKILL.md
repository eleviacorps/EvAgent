# Structure Outlining

## Steps
1. Define the core message: what is the single most important thing the reader should remember?
2. Identify audience and purpose: inform, persuade, instruct, or entertain?
3. Choose structure: inverted pyramid (news), problem-solution (marketing), chronological (tutorials)
4. Draft headings: each heading should advance the argument or narrative
5. Fill key points under each heading: 2-5 bullet points of what each section must cover
6. Review flow: does each section logically lead to the next? Are there gaps or redundancies?

## Examples
- Blog post: Hook → Problem → Solution → Evidence → Actionable Tips → Conclusion
- Technical doc: Overview → Prerequisites → Step-by-Step → Troubleshooting → Reference
- White paper: Executive Summary → Problem → Current Landscape → Approach → Results → Recommendations

## Anti-patterns
- Writing without an outline (leads to rambling, disorganized content)
- Too many sections (overwhelming)
- Sections that don't support the core message
- Starting with detail instead of establishing context
