# Cross-Reference Building

## Steps
1. Identify key concepts across different domains or subjects in your knowledge base
2. Map shared underlying principles (e.g., feedback loops appear in biology, economics, engineering)
3. Note analogies and isomorphic structures (same pattern, different domain)
4. Create bidirectional links: if concept A relates to concept B, note the relationship both ways
5. Tag shared concepts with cross-domain labels for easy retrieval
6. Periodically review your cross-reference graph for new connections as you learn more

## Examples
- Feedback loops: "Negative feedback in homeostasis (biology)" ↔ "Negative feedback in amplifier circuits (electronics)" ↔ "Negative feedback in economics (price mechanism)"
- Network effects: "Metcalfe's law (tech)" ↔ "Epidemiology R0 (disease spread)" ↔ "Social proof (psychology)"
- Optimization: "Gradient descent (ML)" ↔ "Hill climbing (algorithms)" ↔ "Evolution (biology)"

## Anti-patterns
- Forcing connections where none exist (superficial analogies)
- Only connecting within the same domain (misses interdisciplinary insight)
- Creating connections but not reviewing or maintaining them
- Assuming a connection in one direction implies it works in the other
